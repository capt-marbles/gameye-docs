This guide defines how existing docs content transitions from Archbee to Starlight while preserving canonical URL quality.

## Migration Rules

- Keep lowercase, hyphenated canonical slugs.
- Replace opaque/random prefixes with readable routes.
- Ensure one-hop redirects only.
- Preserve intent and internal link graph for high-value docs pages.

## Current GAM-15 Implementation

- Legacy sitemap is crawled from `https://docs.gameye.com/sitemap.xml`.
- Content is converted from Archbee `__NEXT_DATA__` payloads into MDX pages by section.
- Redirects are generated in `/redirects/legacy-redirects.mjs` and wired in `astro.config.mjs`.
- Sidebar navigation is auto-generated per top-level section to keep migrated pages visible.

## Migration Command

Run from the docs project root:

```bash
npm run migrate:archbee
```

## Migration Artifacts

- URL inventory: `/migration/gam-15-legacy-url-inventory.csv`
- Redirect map: `/migration/gam-15-url-normalization-map.csv`
- Generation audit: `/migration/gam-15-generation-results.csv`

## Validation Checklist

- `npm run migrate:archbee` completes without failures.
- `npm run build` completes and emits both canonical routes and legacy redirect routes.
- Legacy random-prefix URLs resolve via redirects to clean canonical paths.
