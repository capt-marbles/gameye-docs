Regions are the primary topology layer for placing sessions close to players while maintaining operational resilience.

## What a Region Represents

A region is a geographic routing boundary that can include one or more concrete locations/providers. You usually select region first, then Gameye resolves to an available location within that region.

## Common Workflows

- Start rollout in one primary region.
- Add a secondary region for fallback capacity.
- Route users by latency or geography from matchmaking.

## Operational Guidance

- Keep at least one fallback region for major player populations.
- Validate image availability in each target region before rollout.
- Monitor session health and failure rates region-by-region during launches.
