This guide gives a practical day-to-day workflow for teams running multiplayer server sessions on Gameye.

## Standard Workflow

1. Build and push a container image for your server.
2. Verify image tags and available locations.
3. Launch sessions through API or dashboard.
4. Observe status/logs and troubleshoot unhealthy sessions.
5. Stop sessions and collect artifacts when complete.

## Operational Recommendations

- Use labels for `environment`, `game_mode`, and `build` to simplify filtering.
- Use stable version tags for production rollouts (`v1.2.3`) instead of mutable tags.
- Keep at least one fallback location configured for each primary region.
- Capture `identifier` values from API errors for support escalation.
