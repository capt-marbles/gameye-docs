This page defines the versioning approach required by GAM-14.

## Versioning Model

- Current docs stream is `v1` (default).
- Future breaking changes will be published under explicit version namespaces.
- Archived versions remain readable and linkable for support continuity.

## Planned URL Pattern

- Current docs: `/...`
- Versioned docs (future): `/v2/...`
- Archived docs (future): `/v1/...`

## Release Cadence

- Patch docs updates: continuous.
- Minor structure updates: weekly release window.
- Major IA/version shifts: milestone-based with redirect review.

## Change Governance

- Changelog entry required for every docs release.
- Canonical/redirect checks required before major restructures.
