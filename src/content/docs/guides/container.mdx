A container image is the deployable unit Gameye launches when you create a session.

## What It Includes

- Your server executable and runtime dependencies.
- Startup command/arguments.
- Network port exposure needed by your game server.
- Environment configuration consumed at runtime.

## In Practice

When a session starts, Gameye pulls the requested image tag, schedules it in an available location, and exposes connectivity metadata (host/port) through API response payloads.

For build workflow details, see [Working with Docker](/guides/working-with-docker).
