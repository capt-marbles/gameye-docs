### Purpose

Nodes are the machines that run containers.

### What you see on this screen

A table that typically includes:

**Name****IP Address****Available **(Yes/No)** Type****CPU **(core count)** RAM****Last update****Region****State **(Alive/Dead)** Node Pool******![Legacy documentation image](https://archbee-image-uploads.s3.amazonaws.com/_em5a3TH-KfZcBHkdKiWt/0nR64BUH0Rr3BksG8zMQ-_image.png)

### Tips **Alive/Dead** is based on the freshness of the node’s last update.

If you can see nodes but can’t edit them, you likely have read-only access.
