Docker is the packaging format Gameye uses to run dedicated server workloads consistently across regions.

## Why Docker Matters in Gameye

- Reproducible runtime behavior across environments.
- Versioned rollouts through image tags.
- Fast rollback by reusing known-good tags.

## Minimum Team Baseline

- Build images in CI, not local-only workflows.
- Tag releases with immutable versions.
- Store images in a registry accessible to your Gameye organization.

If you are new to containerization, start with [Working with Docker](/guides/working-with-docker).
