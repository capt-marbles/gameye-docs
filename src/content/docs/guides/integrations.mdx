Most teams integrate Gameye with three systems: game server build pipelines, matchmaking/orchestration logic, and telemetry tooling.

## Common Integration Targets

- **Game engine runtimes**: Unity and Unreal dedicated server builds.
- **CI/CD pipelines**: image build, scan, publish, and rollout gates.
- **Matchmaking services**: pass target region/mode and map metadata into session creation.
- **Observability systems**: collect session status and logs for incident response.

## Recommended Integration Sequence

1. Automate Docker image publishing from CI.
2. Validate image tags before launch using API checks.
3. Wire session lifecycle calls into matchmaking flow.
4. Forward runtime logs to your existing monitoring stack.
