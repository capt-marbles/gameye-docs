All Gameye Session API endpoints require bearer token authentication.

## Header Format

```txt
Authorization: Bearer YOUR_TOKEN_HERE
```

## Example

```http
GET /session HTTP/1.1
Host: api.gameye.io
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## Token Handling Guidelines

- Store tokens in server-side environment variables.
- Do not expose tokens in client-side code.
- Rotate tokens periodically based on your security policy.
- Use sandbox credentials for testing workloads.

## Common Auth Errors

Authentication failures return `401 Unauthorized`.

Typical causes:

- Missing token
- Invalid token format
- Expired token
- Token not scoped for requested action

```json
{
  "statusCode": 401,
  "code": "UNAUTHORIZED",
  "message": "Authentication failed",
  "details": "Invalid bearer token",
  "path": "/session",
  "identifier": "abc123",
  "timestamp": "2023-04-01T12:00:00Z"
}
```
