Gameye runs your dedicated game servers as Docker images. A reliable image workflow is the foundation of stable runtime behavior.

## Build Checklist

- Start from a versioned base image.
- Keep runtime dependencies explicit and pinned.
- Expose required game/server ports in the image definition.
- Add a health check where possible for faster diagnostics.

## Tagging Strategy

Use two tag types:

- Immutable release tags: `game-server:v1.8.4`
- Environment tags: `game-server:staging`, `game-server:production`

Use immutable tags for session launches in production. Keep environment tags for convenience, not for strict rollout guarantees.

## Publish and Validate

1. Push the image to your registry.
2. Confirm the tag exists with [Tag verification](/api/tag-verification).
3. Run a canary session in one location.
4. Promote to full rollout after log and status checks pass.
