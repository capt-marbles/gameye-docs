### Purpose

Active Containers shows containers that are currently running (or starting).

### What you see on this screen

A table that typically includes:

**Session ID **(container identifier)** Application****Organization **(may be present)** Hostname **and** IP **(where the container is running)** Ports **(mappings shown per container)** Status **(for example Running / Starting)** Region****Node Pool****Created At**(relative time)

![Legacy documentation image](https://archbee-image-uploads.s3.amazonaws.com/_em5a3TH-KfZcBHkdKiWt/5JhyyM5hPpcN_hsO89Qsc_image.png)

### Common actions

Depending on your access, you may be able to:**Start **: Start a new container.** Stop container**: Stop a running container.

### How to: start a container

Open **Active Containers**.

Select **Start**.

Fill in the required options (typically application, region/node pool, and ports).

Submit.

### How to: stop a container

Find the container row.

Select **Stop container**.

Confirm.

### Tips

If you can’t see **Start**, your account may be read-only for container operations.

A container may show **Starting** for a short period after creation.
